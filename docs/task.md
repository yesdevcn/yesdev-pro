# 2.4 任务和工时登记

任务是最小的工作项，工时是可以对研发产能进行量化。  

![](./img/1652413728892.jpg)  
 

# 2.4.1 员工工时

## 工作组工时登记配置

针对于工作组，你可以开启/关闭工时登记。关闭工作组的工时登记后，整个工作组将取消工时登记，重新开启后恢复。  

![](./img/20220825214203.png)  

在同一个工作组内，对于不需要登记工时的成员，可以设置为隐藏名单。即可排除该成员的工时登记。  

![](./img/20220825214243.png)  

> 温馨提示：工作组的管理员，才能权限开启/关闭工时登记，以及隐藏名单。      

## 每日/每周的工时登记 

在【工时登记】，你可以按个人、按部门、按工作组进行工时的查看、统计以及快速登记。  
![](./img/task_20220825214625.png)  

你也可以把任务关联到项目、需求或问题。  
![](./img/WX20220513-115547.png)  

## 实际工时登记

在任务栏，点击新建任务，弹出弹窗；点击【更多设置】，可填写实际工时；同步可以填写父任务ID，关联任务；点击确认按钮，新建任务完成，系统会自动汇总任务工时数据；  
![](./img/2024-08-02-01.png)  

![](./img/2024-08-02-02.png)  

任务面板可查看具体的实际工时，顶部工时统计汇总实际工时总工时和任务总个数；仪表盘增加的黄色圆环为实际工时的汇总统计。  

![](./img/2024-08-02-03.png)  

## 工作负荷分析

在 工作台-我的任务，可以查看自己的工作负荷分析统计图表，类似如下：   

![](./img/20241010-183905.png)  

此外，在【效能】-【数据报表 】-【工作负荷分析】，管理员可以指定成员、以及时间范围的数据查询。

# 2.4.3 任务常用操作

## 子任务

通过设置任务的父任务，可以维护多层级的子任务。  

![](./img/task_multi_sub_list.png)    

## 任务里程碑

通过设置开启 任务里程碑，可以指定里程碑。开启任务里程碑后，将会显示【里程碑图标】。  

![](./img/task_list_with_milestone.png)  

在项目里程碑中，也会自动抓取对应的任务里程碑，进行统计和显示。  

![](./img/project_task_milestone_demo.png)  

## 同步创建任务

** 1、同步创建需求任务 **   

![](./img/20240813-120541.png)  

在创建新需求时，勾选：【同步创建任务】，YesDev会自动创建以下任务：

 - 新建任务（1）自动创建新任务给需求创建人；
  - 自动填充任务标题、任务计划完成时间（同步需求计划完成时间）等；
  - 自动关联需求、和需求所在的项目等；

 - 新建子任务（2）（若需求指定负责人，且不是创建人）自动创建新任务给需求负责人； 
  - 自动填充任务计划开始、计划完成时间（同步需求计划开始、计划完成时间）；
  - 任务状态为：待办，并且以上述创建人的任务为父任务；

自动创建的需求任务，类似如下：  

![](./img/20240813-120905.png)  

** 2、同步创建问题任务 **   

在创建新问题时，勾选：【同步创建任务】，YesDev会自动创建以下任务：

 - 新建任务（1）自动创建新任务给问题创建人；
  - 自动填充任务标题、任务计划开始及结束时间为当天、任务描述等；
  - 自动关联问题、需求、项目等；

 - 新建子任务（2）（若需求指定负责人，且不是创建人）自动创建新任务给问题负责人；   
  - 任务状态为：待办，任务类型：修复缺陷，并且以上述创建人的任务为父任务；

问题自动任务创建效果，类似需求任务。  

## 批量操作任务

在项目任务列表、需求任务列表、问题任务列表等场景，或在任务查询列表，勾选多个任务后，可以进行批量操作。主要分为三大类批量操作：  

 + 第一类：批量快速复制
    + 仅复制标题
    + 复制标题 + 需求链接

 + 第二类：批量更新
    + 任务负责人
    + 工时评估
    + 计划完成时间
    + 任务状态
    + 任务验收状态
    + 任务类型
    + 关联项目/需求
    + 父任务ID
    + 任务里程碑
    + ……

 + 第三类：批量删除
    + 批量移除关联
    + 批量彻底删除

# 2.4.4 任务高级配置

## 任务工时、权限和延期通知配置

使用企业管理员账号，进入：【企业管理后台】-【系统高级配置】-【任务工时配置】，可以分别设置： 

 + 任务权限设置  
   + 谁可以指派任务（任何人、仅限自己）
   + 谁可以修改任务（任何人、创建人+负责人）
   + 谁可以删除任务（任何人、创建人、禁止删除）

 + 任务工时设置
   + 每天工作时长（单位：小时）
   + 单个任务工时不能超过（单位：小时）
   + 当天任务工时不能超过（单位：小时）
   + 任务完成后多久禁止修改（单位：小时）
   + 任务完成多久后收纳（单位：小时）

 + 延期通知规则
   + 超过多久未完成则提醒
   + 提醒周期（星期一至星期日，多选）
   + 提醒时段（例如工作时段：08:30 ~ 21:00）
   + 最多提醒多少次


![](./img/task-20240530-184229.png)  


## 任务状态扩充

YesDev默认的任务状态只有3个，分别是：待办、进行中和已完成。如果需要更多、更细致的任务状态管理，可以启用和开放更多自定义的任务状态。  

使用企业管理员账号，进入：【企业管理后台】-【系统高级配置】-【全局别名配置】，在【任务】进行自定义设置。  

假设，需要的任务状态流程是：  
```
待开发 -> 开发中 -> 开发完成 -> 待测试 -> 测试通过 -> 待上线 -> 已上线 -> 已关闭
```

那么可以按以下截图进行设置。

![](./img/task-20240530-185149.png)  

> 温馨提示：YesDev系统默认开放3个任务状态（待办、进行中和已完成），同时预留了21个任务状态、分为三个任务阶段！
