# 2.2 工作组

> 温馨提示：在进行项目管理前，请先创建工作组，协作更方便。  

## 2.2.1 创建工作组

工作组是一个虚拟的组织，一个员工可以加入多个不同的工作组，体现了团队协作的灵活性。

因此，在创建新项目之前，我们必须先设置工作组。  

![](http://cd8.okayapi.com/yesyesapi_20210519121353_03b03d408e8affd7c06025a69589ab94.png)  

创建后会在工作组列表中查看到，并且可以在列表中【操作】进行编辑，并可以添加新的成员进群组。  

![](http://cd8.okayapi.com/yesyesapi_20210519121611_9d0da594f98e2bbfe1eb45ff8fedcfd8.png)   

## 2.2.2 工作组将会显示在哪里？

首先，工作组，将会以 **隐藏式** 显示在【项目管理】。意思是指，每个团队成员，在项目管理中，能看到的项目，都是他有权限能看到的项目。  
类似聊天群，只有这个人进群了，他才能看到这个群的聊天消息。这样可以更好地让团队避免不必要信息的干扰和冲击。一起保护程序员。  

![](http://cd8.okayapi.com/yesyesapi_20210519121701_fbb8c3f2da78b379b5a83e31253b5a1c.jpg)  

其次，在相关的项目管理以及工作排期中，也会显示工作组列表和分类。
