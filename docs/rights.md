# 1.3 权限配置

为了更有加效进行项目管理和协作，在开始正式使用前或使用过程中，可以由企业管理员或其他团队成员进行相应的权限配置。  

YesDev常用的权限控制主要有：  
 + **工作组权限**  
 工作组可自由组建，按需添加工作组成员，用于控制项目的查看访问权限。  
 
 + **产品线权限**  
 产品线可自由添加，按需绑定到部门，用于控制需求、问题、PRD等访问权限。  
 
 + **部门权限**  
 由企业管理员后台维护，按需划分成员账号，用于控制日报、周报、任务看板等访问权限。

 + **账号权限**  
 针对成员账号，可以设置企业管理员、上级、普通成员、访问等权限。  

 + **菜单权限**  
 针对不同的成员，分配和开放需要提供的菜单权限。(开发中，敬请期待)  
 
 + **发布权限**  
 分为发布管理员和普通成员两类权限，普通成员只允许在常规发布窗口执行发布操作，发布管理员可以在任意时间段进行发布。  

 + **业务权限**  
 通常而言，业务单元的创建者和负责人拥有编辑、修改和删除的权限。  

只有添加和分配相应的权限后，成员账号才有相应的访问和操作权限。但为方便管理，企业管理员拥有全部权限，不管权限分配与否。    
 
> 💎温馨提示：企业管理员拥有最高权限。
