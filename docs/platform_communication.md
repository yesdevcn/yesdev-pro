# 3.8 团队沟通

在软件项目协作过程中，需要团队进行密切沟通和频繁协作。  

以下YesDev功能和设计将能增强和提升团队沟通的质量和效率。  

## 日报
日报可用于进行每日团队成员的当天工作情况，并提供了智能自动汇总能力，方便快速汇总、汇报和记录。  

![](http://cd8.okayapi.com/yesyesapi_20211006164004_9e4cf38fffe1c8c52aab846b2a5d2850.png)

## 周报

周报是指组织部门为共享单位，同一部门内的人员可以相互查看团队成员的周报，但只有本人才能修改自己的周报。  

![](http://cd8.okayapi.com/yesyesapi_20210601111057_8ae86fee59015b0d4dbe7078237374ad.png)  

编写周报时，YesDev将会结合你本周完成的项目、需求、任务和解决的问题进行自动汇总和生成周报内容。你可以在这基础上进行确认、修改和补充，然后保存或发送邮件给团队。  
![](http://cd8.okayapi.com/yesyesapi_20210601111348_606d36f27ae6c34fac8147169a2afb66.png)   


## 一键复制分享

在项目详情页、需求详情页、问题详情页，可以【复制分享】相应的项目、需求和问题，然后分享到微信、企业微信、钉钉或邮箱等沟通场景。  

### 复制分享项目

项目分享的信息包括：  
 + 项目名称
 + 项目负责人
 + 项目总工时和进度
 + 项目链接
 + 项目需求列表及其需求负责人和需求状态

例如项目的复制分享，粘贴后类似：  
```
【一个超大的项目】负责人：管理员, 工时：7H/20H，进度：35%
 https://www.yesdev.cn/platform/project/projects-detail?id=136
1、2021-06-01，微信支付对接，需求中，@林晴
2、2021-06-02，商城系统，设计中，@张三    
```


### 复制分享需求

需求分享的信息包括：  
 + 需求标题
 + 需求负责人
 + 计划上线时间
 + 当前状态
 + 需求链接
 + 任务列表及其负责人和任务状态

例如：
```
微信支付对接，负责人：林晴，计划上线：2021-06-01，状态：需求中
https://www.yesdev.cn/platform/requirements/reqm-detail?id=651
1、2021-05-31，需求梳理，@管理员，DONE
2、2021-06-02，前端开发和对接，@管理员，TODO
3、2021-06-04，微信支付平台后端对接，@管理员，TODO
```  

### 复制分享PRD

在线PRD演示复制的信息包括：  
 + PRD标题
 + PRD分享链接

例如：  
```
小智OA-PRD管理
https://www.yesdev.cn/prd-YSZH-139.html
```

> 温馨提示：在分享PRD链接时，请分享较短的链接，以便进行权限访问控制。  

### 复制分享问题

问题分享的信息包括：  
 + 问题类型
 + 问题标题
 + 问题负责人
 + 问题状态
 + 问题链接

例如：  
```
#Bug，账号合成，负责人：管理员 - 中
https://www.yesdev.cn/platform/problem/problemDetail?id=680
```

## 项目汇报邮件

在项目详情页，可以生成邮件，然后YesDev会结合项目的最新信息自动汇总成相应的项目邮件。  

项目汇报邮件包括的信息有：  
 + 项目简报
 + 项目概况
 + 项目需求进度
 + 项目排期
 + 本周已完成
 + 下周任务计划
 + 落款

![](http://cd8.okayapi.com/yesyesapi_20210601113740_dcc4f8a17b23b7413d337ed8b6ca22c5.png)  

此外，YesDev会自动把项目干系人填充到收件人，把项目关注人填充到抄送人。你可以根据自己的需要再进行调整和补充，然后发送邮件，或复制到你的邮件正文。

## 邮件通知

在合适的时机，YesDev会发送邮件通知给个人。主要发送的消息有：  

 + 项目通知
 + 需求通知
 + 问题通知
 + 任务通知

如果需要发送邮件，需要符合和满足以下条件（缺一不可）：  
 + 企业管理员设置并开启了邮箱服务（否则无法发送任何邮件通知）
 + 在个人资料正确填写了自己的邮箱地址（否则无法接收任务邮件通知）
 + 在个人订阅中勾选了相应的消息通知类型（否则无法接收相关主题通知消息）
 + 在指派或操作时，需要勾选邮件通知负责人
 + 符合发送和通知的规则  

邮件通知的示例效果：  
![](http://cd8.okayapi.com/yesyesapi_20210601090924_6639195c1ace2563472216522de1a316.png)   


## 历史版本

历史版本可用于记录重要信息的变更过程和纪录，目前支持：  

 + 支持项目描述的历史版本、版本对比、自动保存新增草稿
 + 支持需求描述的历史版本、版本对比、自动保存新增草稿

例如，在项目详情页，可以查看项目描述的历史版本。  
![](http://cd8.okayapi.com/yesyesapi_20210604225833_2ea5dcb59fe9b151ea2ddf7574fa6dc0.png)  

点击可查看具体某次修改的历史版本内容，并且可以进行版本回滚和恢复。  
![](http://cd8.okayapi.com/yesyesapi_20210604225919_90749b9fea0cefe6e17e9bb0af168cbb.png)  

需求描述内容的历史版本操作与之类似，不再赘述。
