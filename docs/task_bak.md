# 2.4 研发协作

## 2.4.1 工作台
YesDev针对不同的岗位、角色和专业，给每个人提供可自定义的工作台，你可以根据你的日常工作，设置和配置你的工作台。  
![](http://cd8.okayapi.com/yesyesapi_20210519205006_bd8879ada0cc8356971646bba0b1118b.png)  

通过【自定义工作台】，可以勾选需要的小模块，以及拖拉调整显示的先后顺序。  
![](http://cd8.okayapi.com/yesyesapi_20210519205142_a71de632fdd81e241d6ccc3a243e651e.png)  

## 2.4.2 任务工时登记

项目拆解成需求，然后需求再细分可的若干个可量化、可执行的任务，通过【添加新任务】可以指派新任务。  

需要指定负责任务的负责人、工时以及完成时间等。

![](http://cd8.okayapi.com/yesyesapi_20210519204746_f0f0d724186df43ffd27c8e22c7ae648.png)  

任务初始状态为 TODO，在后续的工作中，负责人需要根据实际情况更新任务的状态，才能反映项目的整体完成情况。

你可以直接添加任务，也可以在某个项目中添加任务，还可以在需求中关联添加任务。YesDev还提供了任务顺延的贴心功能，方便在计划添加紧急需求任务时可以自动顺延已有的任务。  

## 2.4.3 工作排期

当团队的成员评估并进行任务排期后，就可以得到以小组为单位，或以部门团队为单位的工作排期。工作排期，以每周为时间单位，显示每个人每天的任务情况。  
![](http://cd8.okayapi.com/yesyesapi_20210519205500_a2fe2dde6444de78bac0eccda7ba9d16.png)   

除了工作排期，还可以使用任务墙进行人员的任务工作对比。任务是一个可以量化的指标，精确评估任务工时并及时跟进，对项目研发的效率提升以及流程改进有非常重要的意义和帮助。
