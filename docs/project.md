# 2.3 项目管理 All In One

值得注意的是，YesDev中所定义和提倡的项目，是指在一定时间周期内完成的有限需求、任务和问题的集合，对应敏捷开发中的一次迭代或Scrumn的一个Sprint。除此之外，其他事务或技术专项也可以通过项目来进行统一管理和流转。简而言之，项目可以是：  

 + 某个版本迭代的需求  
 + 最近一周或两周的迭代需求  
 + 特定客户的定制化或服务项目  
 + 某个外包项目  
 + 创建项目  
 + 技术专项  
 + 重构项目  
 + 技术管理规范  
 + 前端组件库/后端框架项目  
 + 其他需要管理、流转和推进的事项……

项目有几个明显的特点：  
 + 特点1：有指定的唯一项目负责人，对结果负责  
 + 特点2：有指定的时间周期约束  
 + 特点3：需要完成特定的目标需求  
 + 特点4：有特定的场景以及最终需要输出或交付的内容  
 
# 2.3.1 发起新项目

进入左侧【项目】应用页面，可以通过【+ 发起新项目】，或通过点击顶部【项目】功能模块的【添加+】功能按钮，可以快速创建新项目。  

在创建新项目页面，首先填写相关项目信息，例如：项目名称、项目时间、项目负责人，最为重要的是，通过选择 **工作组** 可以限制当前项目的可见范围。  

![](./img/yesyesapi_20211130150025_f7682c74cff87993e202fbc25cf32469.png)

# 2.3.2 项目协作 
 
发起新项目后，即可在你的团队内部进行项目协作。协作的内容包括但不限于：  

 + 工作项：需求、任务、问题、项目文档、测试计划、敏捷看板、输出物……  
 + 统计类：概览、新手引导、排期表、燃尽图、甘特图、项目脑图……  
 + 通用类：链接、附件、备注、历史变更、知识库……  

![](./img/1652413438523.jpg)  

其中，常用的工作项主要有：项目需求、项目任务和工时、项目问题。      

## 项目需求

项目需求，是指在本次项目中需要迭代完成和需求。项目需求可以来源于需求池中已评审的需求，也可以在创建项目后添加补充新需求，或者在项目过程中重新规则需求。  

![](./img/project-need-20220825-160502.png)  

## 项目任务

项目任务，是指为了完成以上项目需求或其他项目目标而需要执行的工作，例如：技术开发、测试、产品设计、会议沟通等。  
在分配任务的同时，可以评估对应的工时。  

![](./img/project-task-20220825-160843.png)

## 项目问题

项目问题，是指在软件提测后，在功能验收过程发现的Bug缺陷，以及项目发布上线后的线上故障、工单、后续产品优化等。  

![](./img/project-bug-20220825-161118.png)

修改项目名称，项目负责人设定好项目需求、项目问题、项目任务后，由项目成员进行需求、问题和任务的录入和协作，最后100%完成全部的工作项。期间，有项目动态的实时通知和推送提醒。

# 2.3.3 项目管理

项目协作是团队成员之间扁平化的协作，例如产品经理提交一个新需求给开发工程师，又如测试工程师创建了一个Bug并指派给开发工程师。  
另一方面，项目管理则更多是作为项目经理、技术负责人或老板，对整个项目的统筹、计划和对风险进度的控制。  

YesDev提供了以下几种对项目管理的方式。  

## 项目权限

通过工作组控制项目访问权限，支持：私有/保护/公开。 

![](./img/20220825-163050.png) 

对于单个项目内的 需求、任务、问题 权限，可以继续在项目【设置】-【项目内权限】- 选择：项目内公开、仅限负责人、相关干系人。  

 + 项目内公开：默认，跟随工作组权限；
 + 仅限负责人：创建人+负责人+企业管理员；
 + 相关干系人：延伸抄送人员；

![](./img/project_inner_rights_setting.png)  

> 温馨提示：只有项目负责人或创建人，才有权限修改项目信息。通常而言，企业管理员拥有全部权限。      

## 项目锁定

锁定后不可插入新需求、新任务。只有项目创建人或负责人，才有项目锁定和解锁的权限。  

![](./img/project-00.png)  

## 自动收纳

自动收纳项目里已经完成的需求、任务和问题，简化工作。  

![](./img/20220825-163142.png)  

## 项目置顶

可以顶置你重要的项目。  

![](./img/project-top-20220825-162737.png)

# 2.3.6 项目插件组件

项目插件组件，是项目的功能和页面组成部分，可以自由组合使用。  

根据协作和工作的需要，主要分为三类插件组件。  
 + 工作项：和软件开发有关的工作内容，例如：需求、任务、问题、项目文档、测试计划、敏捷看板、输出物等。  
 + 统计图表：根据工作项信息，由系统自动汇总、计算和分析的统计数据，包括图表展示、表格数据等。这一类的插件最大特点都是只读。  
 + 通用插件：数字化办公和通用类项目协作中所需要用到的功能，例如：链接、附件、备注、历史变更、知识库……  

## 工作项

当前工作项，包括但不限于：  

 + 项目集：管理和统筹多个项目。  
 + 需求：需求迭代，项目锁定后不可添加新需求。  
 + 敏捷看板：敏捷任务看板，支持不同维度的划分，可用于每日站会沟通。 
 + 问题：关联的缺陷、工单、优化和故障等问题。  
 + 测试计划：关联的测试计划，以及测试执行情况。  
 + 输出物：不同阶段的制品输出物提交和汇总，支持流程配置。  
 + 任务：任务和工时，支持按需求分组。  
 + 项目文档：项目描述文档，支持导出。  


![](./img/2022-08-25T09-25-19.684Z.png)  

## 统计图表

统计图表，主要有：  

 + 概览：展示该项目的简要信息汇总。  
 + 排期表：自动排期，根据任务评估和进度，实时汇总各人员的工时排期。  
 + 燃尽图：每日燃尽图，有利于及时把控项目延期风险。  
 + 新手引导：第一次接触敏捷开发和项目协作的新手引导。
 + 项目脑图：以脑图方式，宏观汇总和自动梳理项目的各类工作项。
 + 甘特图：七彩甘特图，有利于项目的提前规划和排期。 

![](./img/2022-08-25T09-28-21.256Z.png)  

## 超级甘特图

全新甘特图形式，新增任务分级展示，可自动按照项目/需求/任务的三级层次展示项目量化细节，快速掌握项目总体进度；

多项目全新展示，更清晰明了。

![](./img/2024-04-07-07.png)

## 通用组件

最后一类是项目的通用功能。例如：  
 + 链接：重要的网站链接和测试账号。  
 + 附件：可以支持上传各类文件、图片和压缩包。  
 + 备注：项目下备注和留言，支持@操作。  
 + 历史变更： 项目的历史变更和操作记录。
 + 知识库：支持多层级的在线文档，帮助团队整理知识库。  

![](./img/2022-08-25T09-29-53.107Z.png)

# 2.3.7 外部协作和分享的项目

在YesDev，除了可以管理协作自己团队内部的项目，也可以和外部的团队一起协作分享的项目。   

## 协作邀请
在指定项目，你可以邀请外部的指定团队加入该项目协作，权限可以设置为【编辑】或【只读】。例如邀请甲方客户一起见证项目的推进和成果。  


## 外部协作
邀请外部团队加入项目协作后，即可一起共享项目信息。  

![](./img/2022-08-25T09-48-11.144Z.png)   

基于YesDev结构化的项目信息，你和你的团队，可以更快速进行密切的协作。例如有：自动生成项目汇总邮件、一键复制项目信息、自动生成项目燃尽图，更多实用功能，等你来发现！
