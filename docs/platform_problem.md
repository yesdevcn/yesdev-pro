# 3.7 问题跟踪

问题跟踪可用于管理和跟踪Bug、缺陷、用户反馈建议、用户潜在需求、设计上的不足等问题。  

## 创建新问题
通过顶部菜单【添加】-【新问题】，或其他相应的操作入口，可以添加新的问题。  

![](http://cd8.okayapi.com/yesyesapi_20210601090329_e23163399172e15fc25825574a661baf.png)  

添加新问题时，可以设置以下相关信息：  

 + 问题标题
 + 问题描述（支持富文本编辑，支持直接粘贴截图、链接、表格等）
 + 上传文件（支持多个附件）
 + 指派给（负责人）
 + 问题类型（Bug/改进/设计/需求）
 + 优先级（紧急/高/中/低）
 + 关联项目
 + 产品线（用于快速分类）
 + 版本（产品线对应的版本）  

添加时，可以选择是否邮件通知负责人。  

邮件通知类似如下：  
![](http://cd8.okayapi.com/yesyesapi_20210601090924_6639195c1ace2563472216522de1a316.png)  

## 问题管理与流转

在问题管理界面，可以在顶部按产品线快速分类，在上方查看问题统计，并且快速切换问题：所有/指派给我的/我提交的/我关注的。在右侧可以进行快速筛选操作。  
![](http://cd8.okayapi.com/yesyesapi_20210601094944_19327292fbc8cfde259edff274e17e91.png)  

中间区域显示问题列表，在状态下拉菜单选中新状态后，可以立即修改问题状态，进行流转处理。  

> 温馨提示：问题管理默认显示未修复的问题，即包括：待解决、进行中、重开、已解决，不包括：暂缓、不解决、关闭。  



## 问题详情与流转

在问题详情可以查看更详细和丰富的信息，方便技术人员进行问题修复，和测试人跟进解决情况。  

![](http://cd8.okayapi.com/yesyesapi_20210601095615_4954568887478454cbd4af4d1f3d1731.png)  

如果需要进行更具体的问题流转，可以在问题详情页，同时修改：问题状态、指派给、备注内容。  
![](http://cd8.okayapi.com/yesyesapi_20210601101406_811e4ad6ccc81e318155fabe112aa579.png)  

在问题详情页右侧，可以修改问题归因，和查看历史变更。

在问题管理页面，可以通过Excel批量导入新问题。

## 缺陷统计

在【问题管理】-【缺陷统计】，可以查看和统计缺陷的分布情况，并且导出Excel表格。  

![](http://cd8.okayapi.com/yesyesapi_20210601102153_4541cc2cb4df0480a65b533303c5b903.png)  

目前提供的统计表格有：  
 + 按产品统计
 + 按负责人统计
 + 按创建人统计
 + 按项目统计
 + 按需求统计
 + 按优先级统计
 + 按归因统计
 + 按解决时长统计

> 温馨提示：缺陷统计，只统计Bug类型，不统计以下类型：改进、设计、需求。   

在更多操作菜单，可以导出Excel表格。导出表格内，按每个统计表格为一个Sheet进行导出。例如：  

![](http://cd8.okayapi.com/yesyesapi_20210601102440_36071389cbd9d39ac96e650065213536.png)  


## 测试用例

一个测试用例，可以包括以下基本元素：用例标题、用例描述、测试步骤、用例类型、优先级、用例状态、所属的用例库的模块等。

![](http://cd8.okayapi.com/yesyesapi_20210908115122_9a1192407e00bac77bcb2cb99ae2403f.png)  

在YesDev协作工具中，你可以创建多个测试库。每个测试库对应一个产品或某个项目。

![](http://cd8.okayapi.com/yesyesapi_20210908115302_238473596985647007fa9e3e455b4c76.png)  

你可以单个创建测试用例、复制测试用例，也可以使用Excel批量导入测试用例。  

![](http://cd8.okayapi.com/yesyesapi_20210908115352_52593db7c4114270c973cc5ca68829ee.png)  

## 测试计划

测试用例，是静态的测试单元。

要发挥测试用例的价值，需要进行合理的规划，通过测试计划，可以完成指定时间背景下需要实现的功能为测试目标。这是一种计划能力，也是一种团队协作的重要组成部分。

创建测试计划时，需要指定所属用例库。  
![](http://cd8.okayapi.com/yesyesapi_20210912095823_3775322a71122ad433ea69bc5774d693.png)  

一个测试计划，可以规划多个测试用例。计划的边界范围，取决于项目的要求。如果是全新的项目，则需要分批涵盖新功能的全部测试用例。如果是常规的版本迭代，可以把一级用例（即那些必须要通过回归测试的用例）单独维护成一个测试计划，以便在每次发布上线前进行一轮回归测试。

![](http://cd8.okayapi.com/yesyesapi_20210908115504_cc1ef998e39774ffb27b1365b65c219c.png)  

在具体的测试计划里，要关注两个100，分别有：测试用例要100%测试通过；全部缺陷要100%修复。  

![](http://cd8.okayapi.com/yesyesapi_20210908115642_b3cf6c8f791ffee5ae2e8c8e42bc9647.png)  

开始测试：可以快速对未测试完毕的用例继续进行测试。  

![](http://cd8.okayapi.com/yesyesapi_20210908115733_4b604fefd0555192c711f7b307a5aeed.png)  

测试计划可以和项目关联，方便项目进行更全面的质量管理和风险管理。  
![](http://cd8.okayapi.com/yesyesapi_20210908120041_569450d573e682714ac50279a0b5f9e1.png)  


## 测试报告

在测试过程中，要及时进行测试进度的同步和反馈。如果项目周期较长，或者测试所需要的时间较多，应当在下班前或关键的时间节点，或项目里程碑到来前，或者测试进度不理想有延期风险时，应当通过测试报告向上汇报或在团队内部进行同步，或向客户反馈。YesDev提供了自动生成汇总的测试报告，可以直接发送到邮件。  

![](http://cd8.okayapi.com/yesyesapi_20210908115827_8112eb2b7cd1df78c71a25062d94b75c.png)  

自动生成的测试报告内容，类似如下：  
![](http://cd8.okayapi.com/yesyesapi_20210908115914_304c02cf747754a8981ad3c7a7d3a91d.png)  

你可以进行二次修改，也可以直接发送邮件给项目干系人。
